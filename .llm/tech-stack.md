## Language & Runtime

- **Node.js 20+** — primary build/runtime used in CI and local development
- **Astro 5** — static site generator and route renderer
- **TypeScript** — used for Astro config, content schemas, and helpers

## Frontend

- **Astro + React** — Astro owns routing/content; React components power the shared visual system and homepage sections
- **CSS**: a single design-system stylesheet in `public/styles/globals.css`
- **JavaScript**: existing scripts in `public/scripts/` are still served from `public/`
  - `toc.js` handles the interactive table of contents
  - `theme-toggle.js` handles the explicit light/dark theme toggle
- **Storybook** documents the shared React components in isolation

## Hosting & Deployment

- **Vercel** — static site hosting with global CDN
- **GitHub Actions** — CI/CD pipeline (`.github/workflows/go.yml`)
- Trigger: push or PR to `main`, plus push to `preview`
- Deployment builds with `npm ci && npm run build`, then deploys `build/` via `npx vercel`

## Content System

- **Astro content collections** load Markdown directly from the existing root `posts/` and `pages/` directories
- **Zod schemas** validate front matter during the build
- No intermediate database is used in the active build

## SEO & Standards
