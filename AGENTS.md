# Project Context

Before starting any task, always read the context files in the `.llm/` directory:

- `.llm/structure.md` — Project structure and directory layout
- `.llm/tech-stack.md` — Language, dependencies, and tooling
- `.llm/build-and-deploy.md` — Build steps, CI/CD pipeline, and deployment
- `.llm/conventions.md` — Front matter schema, URL structure, code style, and how to keep context files updated
- `.llm/content.md` — Index of all blog posts and static pages

When making changes to the project, update the relevant `.llm/` files to keep them in sync (see `.llm/conventions.md` for the update rules).
